Introduction

Base URL: http://recommender.test

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

submit a user reading interval

Example request:
curl --request POST \
    "http://recommender.test/api/new-read-interval" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"user_id\": 1,
    \"book_id\": 1,
    \"start_page\": 1,
    \"end_page\": 10
}"

const url = new URL(
    "http://recommender.test/api/new-read-interval"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "user_id": 1,
    "book_id": 1,
    "start_page": 1,
    "end_page": 10
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

user_id   integer   

The id of the user. Example: 1

book_id   integer   

The id of the book. Example: 1

start_page   integer   

The start page of the interval. Example: 1

end_page   integer   

The end page of the interval. Example: 10
